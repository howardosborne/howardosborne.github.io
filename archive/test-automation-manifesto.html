Test Automation Manifesto - a Proposal

I recently got asked to come up with a set of simple rules for test automation. Well rather than preach how to test, I thought I would come up with something that could be a bit more consensual. So here is a proposed manifesto:

  1. Automation is development. We therefore apply the same practices to make and maintain our code.
  2. A user interface becomes an API when automating. We treat it like any other API.
  3. We invest in making robust test environments where tests can be run repeatably.
  4. We keep our tests short and sweet.
  5. We keep dependencies between tests to a minimum.
  6. Browsers are wonderful applications, but they are big and slow. We don't test for things through the browser that could have been found in a simpler way.
  7. We use automation for the things that computers do best and use humans for the intelligent stuff. Computers are good at 'that's right/wrong' - humans are good at 'that looks weird'.
  8. The automation graveyard is full of good intentions. We only pursue an automation project if we can apply the above rules.
